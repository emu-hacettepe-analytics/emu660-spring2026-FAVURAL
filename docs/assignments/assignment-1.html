Open Ended Question:

Question:
In the data science workflow why is problem definition considered the most critical first step? Explain using the “Survivorship Bias”.

Answer:
Problem definition is critical because solving the wrong problem leads to incorrect conclusions even if the analysis is technically correct. In World War II where analysts examined bullet holes on returning bombers and suggested reinforcing the areas where most holes appeared. However, statistician Abraham Wald realized that the data only came from planes that survived. The planes that were hit in other areas (such as the engine or cockpit) never returned. Therefore, reinforcing the already damaged areas was incorrect; the vulnerable areas with no bullet holes on surviving planes were actually the places that needed reinforcement. This example demonstrates that correctly framing the problem determines whether the analysis leads to useful decisions.

# Custom function to compute statistics
compute_stats <- function(x) {
  stats <- list(
    mean = mean(x),
    median = median(x),
    variance = var(x),
    IQR = IQR(x),
    minimum = min(x),
    maximum = max(x)
  )
  return(stats)
}
